# パッドグリッド拡張 要件

これは **未実装の要件メモ**。JPad のパッド表示を、現行 12 パッドに加えて 4x4 モードへ拡張するための要件をまとめる。

## 目的

コード進行スケッチ用途を保ちながら、以下を実現する。

- 16 パッドを扱う `4x4` モードを追加する
- 横向き時に、パッドの表示と配列を横向きに合わせるモードを用意する
- 既存の `3x4` モードでも、横向き時の並びを直感的にする
- パッド番号の意味を、縦向き・横向きで崩さない

## 用語

| 用語 | 意味 |
|------|------|
| `3x4` | 現行の 12 パッド構成 |
| `4x4` | 追加予定の 16 パッド構成 |
| パッド番号 | `01` から始まる表示・編集上の番号 |
| 表示順 | 画面上でパッドが並ぶ順序 |
| データ順 | JSON / Preset 内で保持する `PadDefinition.index` |

## 基本ルール

`4x4` では、縦向き・横向きともに **左下を 01、右上を 16** とする。

`3x4` の横向きモードでも同じ考え方を使い、**左下を 01、右上を 12** とする。

つまり、画面上の視覚的な番号配置は常に次のルールで決める。

- 左から右へ番号が増える
- 下から上へ段が上がる
- 一番左下が最小番号
- 一番右上が最大番号

## 4x4 モード

### 表示

4x4 は 16 パッドを表示する。

縦向き:

```text
13 14 15 16
09 10 11 12
05 06 07 08
01 02 03 04
```

横向き:

```text
13 14 15 16
09 10 11 12
05 06 07 08
01 02 03 04
```

横向きでも 4x4 の見た目は正方グリッドを維持する。必要に応じてパッドサイズ、上部バー、下部コントロールの余白を再計算する。

### データ

- 16 個の `PadDefinition` を持つ
- index は `0...15`
- 表示ラベルは `01...16` に対応する
- 既存 12 パッドプリセットを 4x4 に変換する場合、`13...16` は空パッドとして追加する

## 3x4 モード

現行の 12 パッド構成は維持する。

縦向き:

```text
10 11 12
07 08 09
04 05 06
01 02 03
```

横向き:

```text
09 10 11 12
05 06 07 08
01 02 03 04
```

横向きでは、現在の `6x2` 表示ではなく、`4x3` 表示を基本候補とする。これにより、左下 01 → 右上 12 のルールを維持する。

## 横向き配列モード

横向き時に、パッドの表示と配列のみ横向きに合わせるモードを追加する。

このモードで変わるもの:

- パッドの列数・行数
- パッドの表示順
- パッド内ラベルの位置調整
- パッドサイズ

このモードで変えないもの:

- プリセットの中身
- パッド index
- コード情報
- TRANSPOSE / SLIDER の値
- MIDI OUT / MIDI IN 設定

## 保存要件

グリッドモードはプリセット単位で持つか、アプリ全体設定で持つかを決める必要がある。

現時点の推奨:

- `3x4 / 4x4` はプリセット単位
- 横向き配列モードはアプリ全体設定

理由:

- 4x4 はパッド数そのものが変わるため、プリセット構造に関係する
- 横向き配列は表示方針なので、ユーザーの操作環境に近い

## JSON / 互換

### 既存 JSON 読み込み

- `pads` が 12 個なら `3x4`
- `pads` が 16 個なら `4x4`
- グリッドモード未記載の場合は `pads.count` から推定する

### EXPORT / IMPORT

- JSON にはグリッドモードを保存する
- 既存 JSON との互換のため、未記載時は `3x4` 扱い
- 16 パッドの JSON を古いバージョンへ戻す互換は保証しない

### AIRDROP

- `AIRDROP` は既存のプリセット共有経路を使う
- 中身の preset JSON にグリッドモードが含まれればよい

## UI 要件

### 設定場所

候補:

- プリセット編集画面
- Presets 画面のセット設定
- 設定画面

現時点の推奨:

- `3x4 / 4x4` は Presets 画面またはプリセット編集側
- 横向き配列モードは設定画面

### 表示

- 4x4 ではパッドが小さくなるため、ラベルのフォント縮小が必要
- iPhone 横向きでは下部コントロールとの競合に注意する
- iPad では余白が出すぎないよう、最大サイズ制限を別に持つ

## 実装上の注意

既存コードでは、レイアウト計算と演出が 12 パッド前提を持っている。

影響が出る箇所:

- `JChordPadLayout`
- `MainViewModel.displayOrder`
- `PadGridLayoutGeometry.padCount`
- `PadPerformanceEffectEngine`
- `PadPerformanceScanPattern`
- パッド編集画面
- `Preset` / `PadDefinition` の JSON 読み書き

4x4 対応では、まず `padCount` を固定値ではなく layout / preset 由来にする必要がある。

## 実装フェーズ案

1. 3x4 横向き表示を `4x3` にする
2. 表示順を左下 01 → 右上 12 に統一する
3. `PadGridLayoutGeometry` を可変 padCount にする
4. `4x4` の表示だけ作る
5. `Preset` の 16 パッド保存に対応する
6. 4x4 の編集・IMPORT / EXPORT / AIRDROP を確認する
